Functions 개요

Oracle Functions 는 fully managed, multi-tenant, highly scalable, on-demand, Functions-as-a-Service platform입니다. 엔터프라이즈 급 Oracle Cloud Infrastructure 를 기반으로하며 Fn Project 오픈 소스 엔진으로 구동됩니다. Oracle Functions(주로 줄여서 Functions)를 사용하면 비즈니스 요건에 맞춘 코드 작업을 수행할 수 있습니다..

Oracle Functions 의 서버리스 및 탄력적 아키텍처는 수행 할 인프라 관리 또는 소프트웨어 관리가 없음을 의미합니다. compute 인스턴스를 프로비저닝하거나 유지 관리필요가 없으며, 운영 체제 소프트웨어 패치 및 업그레이드가 자동으로 적용됩니다. Oracle Functions는 앱의 고가용성, 확장성, 보안 및 모니터링을 간단히 보장합니다. Oracle Functions를 사용하면 Java, Python, Node, Go 및 Ruby로 코드를 작성할 수 있습니다(고급 사용 사례의 경우, 직접 작성한 Dockerfile, Graal VM등을 가져올 수 있습니다.). 그런 다음 코드를 배포하고 직접 호출하거나 이벤트에 대한 응답을 통하여 코드를 실행하고, 실행 중에 소비된 리소스에 대해서만 요금이 청구됩니다.

Oracle Functions 는 Fn Project를 기반으로 합니다. Fn Project 는 클라우드 또는 온-프레미스 어디에서나 실행할 수있는 오픈 소스 컨테이너 네이티브 서버리스 플랫폼입니다. Fn Project 는 사용하기 쉽고 확장 가능하며 기대한 성능을 발휘합니다. Fn Project 의 오픈 소스 배포판을 다운로드하여 설치하고 로컬에서 함수를 개발 및 테스트 한 다음 동일한 도구를 사용하여 해당 함수를 Oracle Function에 배포 할 수 있습니다.

Console, CLI 및 REST API를 사용하여 Oracle Functions에 액세스 할 수 있습니다 . CLI를 사용하거나 서명된 HTTP 요청을 통하여 Oracle Functions에 배포 한 기능을 호출 할 수 있습니다 .

Oracle Functions 는 Oracle Cloud Infrastructure Identity and Access Management (IAM) 와 통합되어 기본 Oracle Cloud Infrastructure ID 기능으로 쉽게 인증 할 수 있습니다. Overview of Oracle Cloud Infrastructure Identity and Access Management를 참조하십시오 .

Oracle Functions를 신속하게 설정하고 실행 하려면 Quick Start Guides 참조하십시오 .

Oracle Cloud Infrastructure 에 액세스하는 방법

Console(브라우저 기반 인터페이스) 또는 REST API를 사용하여 Oracle Cloud Infrastructure에 액세스 할 수 있습니다 . Console 및 API에 대한 지시 사항은 이 안내서 전체의 주제에 포함되어 있습니다. 사용 가능한 SDK 목록은 Software Development Kits and Command Line Interface를 참조하십시오 .

Console에 액세스하려면 supported browser를 사용해야 합니다. 이 페이지 상단의 Console 링크를 사용하여 Sign-in 페이지로 이동할 수 있습니다. 클라우드 tenant, 사용자 이름 및 비밀번호를 입력하라는 메시지가 표시됩니다.

나머지 API를 사용하는 방법에 대한 일반적인 내용은 REST APIs를 참조하십시오 .

이벤트로 자동화 만들기

이벤트 유형, 규칙 및 조치를 사용하여 Oracle Cloud Infrastructure 자원의 상태 변경을 기반으로 자동화를 작성할 수 있습니다 . 자세한 정보 Overview of Events를 참조하십시오 .

다음 Oracle Functions 자원은 이벤트를 발생합니다.

·        applications

·        functions

다른 서비스의 이벤트가 Oracle Functions의 함수를 호출하도록 할 수도 있습니다 . Invoking Oracle Functions from Other Oracle Cloud Infrastructure Services를 참조하십시오 .

자원 식별자(Resource Identifiers)

대부분의 Oracle Cloud Infrastructure 리소스 유형 에는 Oracle Cloud ID (OCID)라는 고유 한 Oracle 할당 식별자가 있습니다. OCID 형식 및 기타 리소스 식별 방법에 대한 자세한 내용은 Resource Identifiers를 참조하십시오 .

인증(Authentication) 및 권한(Authorization)

Oracle Cloud Infrastructure의 각 서비스는 모든 인터페이스 ( Console , SDK 또는 CLI 및 REST API)에 대한 인증 및 권한 부여를 위해 IAM 과 통합됩니다 .

조직의 관리자가 groups 을 설정해야 합니다 , compartments 및 policies 어떤 사용자가 어떤 서비스, 어떤 리소스 및 액세스 유형에 액세스 할 수 있는지 제어합니다. 예를 들어, policies는 새 사용자 생성, 클라우드 네트워크 생성 및 관리, 인스턴스 시작, 버킷 생성, 객체 다운로드 등을 제어 할 수 있는 사용자 제어합니다. 자세한 내용은 Getting Started with Polices를 참조하십시오 . 서로 다른 서비스 각각에 대한 정책 작성에 대한 자세한 내용은 Policy Reference를 참조하십시오 .

회사 소유 의 Oracle Cloud Infrastructure 리소스 를 사용해야하는 일반 사용자 (관리자가 아님) 인 경우 관리자에게 문의하여 사용자 ID를 설정하십시오. 관리자는 어떤 compartment를 사용해야하는지 확인할 수 있습니다.

오라클 Functions 가용 및 한계

tenancy에서 생성 가능한 최대 자원 갯수는:

·        10 개의 applications

·        20 가지 functions

적용 가능한 제한 목록 및 제한 증가 요청에 대한 지침 은 Service Limits 을 참조하십시오 .

함수 (기능의 요청 페이로드)에 보낼 수있는 최대 데이터 양은 6MB입니다. 함수가 요청에 응답하여 리턴 할 수있는 최대 데이터 양 (함수의 응답 페이로드)은 6MB입니다. 이러한 제한은 고정되어 있으며 변경할 수 없습니다.

일부 다른 Oracle Functions 기능 및 제한도 고정됩니다. 그러나 변경할 수 있는 값도 있습니다. Changing Oracle Functions Default Behavior 참조하십시오 .
